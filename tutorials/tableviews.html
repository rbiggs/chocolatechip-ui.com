Tableviews

Tableview are the main artifice by which you output data in your app. ChocolateChip provides a number of pre-configured layouts for tableviews making it merely a matter of you deciding which one to use.

There are two ways in which tableview are display: from edge to edge or inset from the sides with rounded corners. The inset style is called "grouped" There is not functional difference between these two styles. It is just a question of taste. Of course, if you have a lot of data that you need to put into a tableview, an edge to edge style would make for since. If your data is sparse you might want to use the grouped style.

Note: ChocolateChip-UI handles all celltitles and cellsubtitles so that if their content exceeds the available space they get truncated with an ellipsis automatically. You don't have to do anything.

 

Basic Tableview with Celltitles:

The simplest tableview has tablecell with just celltitles

Example:

<tableview>
    <tablecell>
        <celltitle>People</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Places</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Things</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Animals</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Food</celltitle>
    </tablecell>
</tableview>

Basic Tableview with Indicators:

Disclosure Indicator

Tablecells can have various indicators which signal to the user that touching the cell will cause something to occur. The most common indicator is "disclosure". It looks like a right facing caret. It's purpose is to indicate that touching the tablecell will result in navigation to another screen. To output a disclosure indicator, just put *ui-implements="disclosure" on the tablecell. To learn how to have the disclosure implement navigation to another screen where there are more options for the user to choose from, refer to the tutorial for List Navigation. The disclosure indicator is used to create what are know as drilldown lists because they allow the creation of applications with deeply nested navigation screens.

Detail-disclosure Indicator:

Another type of indicator is the detail-disclosure. This is similar to the disclosure indicator but is surrounded by a circular border. This is used to indicate that touching the tablecell will lead to another screen where there will be a screen of details about the item the user chose.

Add Indicator

There is also an "add" indicator. Touch a sell with this indicator will trigger some mechanism for the user to add something. As a developer its your job to figure out what and how you will add. You'll need to attach an even to any tablecell with an add indicator to trigger the addition. You could use a selector/delegate combo like this:

$("#myTableView").delegate("tablecell[ui-implements=add]", "touchstart", additionFunction);

Delete Indicator

The delete indicators get output when enabling deletion of tablecells. Refer to $.UIDeleteTableCell in the documentation of ChUI.js.

Variations on Indicators

ChocolateChip-UI offers some variations on these indicators based on the styles that Apple uses to draw user attention to them. Just add a class of "green" to the tablecell and you will get a green version of the detail-disclosure indicator and add indicator. If you want, you could create a new type of indicator appropriate for your needs or create a variation on these. Refer to the documentation on indicators in ChUI.css. The following markup shows how to implement the common tablecell indicators.

Example:

<tableview>
    <tablecell>
        <celltitle>People&hellip;</celltitle>
    </tablecell>
    <tablecell ui-implements="disclosure">
        <celltitle>Places</celltitle>
    </tablecell>
    <tablecell ui-implements="disclosure">
        <celltitle>This is an example of a very long item that will be truncated because it does fit.</celltitle>
    </tablecell>
    <tablecell ui-implements="detail-disclosure">
        <celltitle>Food</celltitle>
    </tablecell>
    <tablecell ui-implements="detail-disclosure" class="green">
        <celltitle>Food</celltitle>
    </tablecell>
    <tablecell ui-implements="add">
        <celltitle>Food</celltitle>
    </tablecell>
    <tablecell ui-implements="add" class="green">
        <celltitle>Food</celltitle>
    </tablecell>
</tableview>

Basic Tableview with Icons

Another type of tableview layout consists of celltitles with application icons on the left. To indicate that tablecells will display icons, put ui-usage="icon" on them. The images for an icon should be 29 pixels by 29 pixels. Insert the images for the icons before the celltitle tag as in the code below:

Example:

<tableview>
    <tablecell ui-usage="icon">
        <img src="images/ipod.png">
        <celltitle>People&hellip;</celltitle>
    </tablecell>
    <tablecell ui-usage="icon" ui-implements="disclosure">
        <img src="images/phone.png">
        <celltitle>Places</celltitle>
    </tablecell>
    <tablecell ui-usage="icon" ui-implements="disclosure">
        <img src="images/mail.png">
        <celltitle>This is an example of a very long item that will be truncated because it does not fit.</celltitle>
    </tablecell>
    <tablecell ui-usage="icon">
        <img src="images/safari.png">
        <celltitle>Animals</celltitle>
    </tablecell>
    <tablecell ui-usage="icon" ui-implements="detail-disclosure">
        <img src="images/sms.png">
        <celltitle>Food</celltitle>
    </tablecell>
    <tablecell ui-usage="icon" ui-implements="add">
        <img src="images/store.png">
        <celltitle>Stuff</celltitle>
    </tablecell>
</tableview>

Basic Tableview with Images:

Often you may need to display a regular image in a tablecell to display some graphic information about that particular item. To output images in tablecells give the them this attribute: ui-usage="image". Images should be 55 pixels by 55 pixels. Insert images before the celltitle tag as in the code below.

<tableview>
    <tablecell ui-usage="image">
        <img src="images/blues.png">
        <celltitle>Places</celltitle>
    </tablecell>
    <tablecell ui-usage="image">
        <img src="images/classical.png">
        <celltitle>This is an example of a very long item that will be truncated because it does not fit.</celltitle>
    </tablecell>
    <tablecell ui-usage="image">
        <img src="images/itunes.png">
        <celltitle>Animals</celltitle>
    </tablecell>
    <tablecell ui-usage="image">
        <img src="images/killers.png">
        <celltitle>Food</celltitle>
    </tablecell>
</tableview>

Basic Tableview with UIButtons in Tablecells:

You can have UIButtons or switchcontrols in tablecells. Just put ui-usage="uibutton" on the tablecell.

Example:

<tablecell>
    <celltitle>People</celltitle>
    <switchcontrol class="off" id="sleepSwitch">
        <label ui-implements="on">ON</label>
        <thumb><thumbprop></thumbprop></thumb>
        <label ui-implements="off">OFF</label>
        <input type="checkbox" value="ZZZZZZZZ!">
    </switchcontrol>
</tablecell>
<tablecell ui-implements="titleWithuibutton">
    <celltitle>Places</celltitle>
    <switchcontrol class="on" id="eatSwitch">
        <label ui-implements="on">ON</label>
        <thumb><thumbprop></thumbprop></thumb>
        <label ui-implements="off">OFF</label>
        <input type="checkbox" value="Yummy!">
    </switchcontrol>
</tablecell>
<tablecell ui-implements="celltitle-with-uibutton">
    <celltitle>Things</celltitle>
    <uibutton>
        <label>View</label>
    </uibutton>
</tablecell>
<tablecell ui-implements="celltitle-with-uibutton">
    <celltitle>Animals</celltitle>
    <uibutton>
        <label>See</label>
    </uibutton>
</tablecell>
<tablecell>
    <celltitle>Food</celltitle>
</tablecell>
</tableview>

This will produce the following:

Tableheader

You can provide a tableview with a tableheader, which can be a title for the table or some introductory information about the tableview. This is an optional tableview element. To provide a tableview with a tableheader, place it as the first element in the tableview.

Example:

<tableview>
    <tablehader>
        A basic tableheader here to announce what this tableview is about.
    </tableheader>
    <tablecell>
        <celltitle>People</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Places</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Things</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Animals</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Food</celltitle>
    </tablecell>
</tableview>

Tablefooter

You can provide a tableview with a tablefooter, which can be a summary about the tableview. This is an optional tableview element. To provide a tableview with a tablefooter, place it as the last element in the tableview.

Example:

<tableview>
    <tablecell>
        <celltitle>People</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Places</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Things</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Animals</celltitle>
    </tablecell>
    <tablecell>
        <celltitle>Food</celltitle>
    </tablecell>
    <tablefooter>
        A quick summary about all of the items in this list.
    </tablefooter>
</tableview>
